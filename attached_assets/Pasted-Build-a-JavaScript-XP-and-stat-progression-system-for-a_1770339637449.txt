Build a JavaScript XP and stat progression system for a gamified productivity app.
The system must follow these exact mechanics:
CORE STRUCTURE
Stats:
strength
cardio
knowledge
discipline
Each stat levels independently. There is also a global level based on total XP.
BASE XP RULES
Strength Task XP:
Full completion: 10 XP to strength
Partial completion: 50% XP (5 XP)
Failed: 0 XP
Cardio Task XP:
Full completion: 8 XP to cardio
Partial completion: 4 XP
Failed: 0 XP
Knowledge Task XP:
Full: 6 XP
Partial: 3 XP
Discipline Daily Bonus:
Completing Minimum Viable Day gives 5 XP to discipline
MINIMUM VIABLE DAY (MVD)
Minimum Viable Day is achieved when:
At least 1 physical task (strength OR cardio) AND
At least 1 mental task (knowledge OR discipline habit)
If MVD achieved:
+5 discipline XP bonus
Streak +1
If NOT achieved:
Streak resets to 0
Apply penalty (see below)
PENALTY RULES
If user misses MVD:
Lose 3% of current total XP
Cannot drop below previous level threshold
Streak resets
If user skips 3 days in a row:
Additional -5 strength XP
Additional -5 cardio XP
ANTI-INSTANT LEVEL SYSTEM
XP required to level up must scale exponentially:
Formula: XP Required = 100 * (level ^ 1.5)
Example: Level 1 → 100 XP Level 2 → 282 XP Level 3 → 519 XP Level 4 → 800 XP
This prevents leveling up from one workout.
PARTIAL COMPLETION LOGIC
If task completion percentage is provided (0–100):
XP Earned = BaseXP * (completionPercentage / 100)
If completion < 30%: No XP awarded.
STAMINA PROGRESSION RULE
Stamina increases automatically every time:
Strength increases by 10 total XP OR
Cardio increases by 10 total XP
Stamina increase formula: Stamina += 1 per 10 combined physical XP
RETURN FUNCTIONS NEEDED
calculateTaskXP(statType, completionPercentage)
applyMinimumViableDay(tasksCompletedToday)
applyPenalty(currentXP, streakMissedDays)
calculateXPRequired(level)
updateStamina(totalStrengthXP, totalCardioXP)
Make this system modular and exportable.
Do NOT allow level-ups from a single task unless player was already near threshold.
